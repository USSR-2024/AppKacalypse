# План: модуль «Документы»

Рабочий план-чеклист. Идём по нему сверху вниз, отмечаем сделанное.
Спецификация — [ТЗ-документооборот.md](ТЗ-документооборот.md). Этот файл — что и в каком порядке делаем.

**Цель:** удобное место для управления документами — от создания и хранения до согласования и сдачи в архив.
**Не цель:** повторить чужой регламент. Регламент, написанный под таск-трекер Shtab, используется как
приёмочный тест, а не как спецификация.

---

## Решения, принятые заранее (не переспрашивать)

| # | Решение | Почему |
|---|---|---|
| 1 | **Центральная сущность — карточка документа.** Файл = её свойство | Модуль документов, а не задачи с вложением |
| 2 | **`tasks` не переиспользуем** как носитель. Домен свой | Иначе тащим шрамы трекерной модели |
| 3 | **ЭДО/ЭЦП — нет.** Подписание бумажное, грузим скан/PDF | Владелец: «эдо нас не касается, интеграций не будет» |
| 4 | **Матрица зовёт ГРУППУ, а не человека** | Иначе увольнение одного юриста ломает все маршруты |
| 5 | **Должность ≠ право доступа.** Два ортогональных измерения | Смешаешь — потом не расплести |
| 6 | **Матрица = данные, редактируемые в UI** | Требование: «редактировать и не лезть в код». У каждой компании своя |
| 7 | **Экземпляр маршрута = снимок шаблона** | Правка матрицы не должна ломать идущие согласования |
| 8 | **Замечание блокирует, комментарий — нет** | Разные последствия (регламент 6.9 vs 6.10). Две явные кнопки |
| 9 | **DS-поля версии резервируем сразу**, DS подключаем или нет — позже | `changesurl`+история не доделываются задним числом (ТЗ §4.6) |
| 10 | **`workspace_id` во всех таблицах** с первого дня | Дописывать в 15 таблиц потом — дорого. Single-tenant = один ws в инстансе |

## Не потерять (грабли, оплаченные заранее)

**Из ТЗ:**
- `key` в DS = `d{id}_v{n}_{hash16}` — хэш в ключе автоматически решает инвалидацию кэша. ТЗ верно зовёт это «источником половины багов»
- Callback DS **всегда** отвечает `{"error":0}` + HTTP 200, иначе DS уйдёт в повторы и бросит файл
- Идемпотентность callback по хэшу — DS шлёт дубли
- `changesurl` + `history` сохранять **с первого дня**
- Автор в черновике пишет без рецензирования, согласующий — принудительно с `trackChanges: true`
- §7.4 (OOXML-реквизиты) — самое хрупкое место. Есть вариант Б: подклеивать при конвертации в PDF. День вместо недели. За границами первого захода

**Из appka:**
- Таблицы в БД **`akc`**, не `task_dispatcher`
- `db:generate` интерактивный → миграции писать руками (SQL + journal + snapshot)
- Миграция без записи в `_journal.json` не применяется
- `tsc --noEmit` проверяет всё дерево, а не стейдж
- Hono матчит роуты по порядку регистрации
- **Репозиторий публичный** — секреты только в `.env` на 158

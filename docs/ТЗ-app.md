# ТЗ — AppKacalypse (свой продукт)

> Разворот от Vikunja+n8n к **собственному приложению**. Дата: 2026-06-12.
> Это источник правды по пользовательской части. Инфраструктура/Gateway — см. `ТЗ-appkacalypse.md`.

## 1. Что строим

AI-диспетчер задач для личной и командной работы. Пользователь ставит задачи
обычным языком (Telegram / приложение), AI разбирает их в структуру и при нехватке
данных доспрашивает. Напоминания — утро/вечер/по запросу, по выбранному каналу.
Всё видно в календаре. Данные и LLM (Qwen) — на своём сервере.

**Что переиспользуем:** GPU-сервер, LLM Gateway + Qwen, Caddy, домены, HTTPS, деплой.
**Что убираем из продукта:** Vikunja, n8n, AppFlowy (были референсами).
**Новое:** свой бэкенд (Node+TS+Hono+Postgres+Drizzle) + PWA (Next.js) + Telegram-бот.

## 2. Роли и доступ

- Глобальные роли: `owner` / `admin` / `member`.
- **Назначение задач: любой пользователь → любому.** Открытая командная модель.
- Править/удалять задачу могут: автор, исполнитель, admin/owner.
- Проект (направление): есть владелец и участники (`lead` / `member`).
- Авторизация — через **Telegram Login** (см. §6). Пароли/email-вход — не в MVP.

## 3. Объект «задача»

| Поле | Значение |
|---|---|
| title, description | текст |
| project | направление или `null` (личная / Inbox) |
| creator, assignee | пользователи (assignee может быть пустым) |
| **status** | `queued` (в очереди) → `in_progress` (выполняется) → `done` (готово); `cancelled`, `archived` |
| **priority** | срочность: `low` / `normal` / `high` |
| **isImportant** | флаг «важная, не забыть» — **отдельная ось** от priority |
| dueAt, remindAt | срок и точечное напоминание |
| source | `app` / `telegram` / `email` / `calendar` / `ai` |

**Inbox:** задача без проекта и неразобранная (`isTriaged=false`) лежит во «Входящих».
Пользователь разбирает → даёт проект или подтверждает «личная» (`isTriaged=true`).

## 4. Способы постановки задач

1. **Приложение** — форма/быстрый ввод (с пикерами проекта, исполнителя, срока).
2. **Telegram-бот** — текст или **пересланное сообщение** → Qwen → структура.
   Если не хватает минимума (что за задача? срок? проект или личная? приоритет?) —
   **бот доспрашивает и ждёт подтверждения**, только потом создаёт.
3. **Из календаря** — быстрое добавление в выбранный день (с пикерами проекта/исполнителя).
4. **Email** — отложено (доп. канал, архитектуру закладываем расширяемой).

AI-разбор уже частично готов в LLM Gateway (`/extract`): intent, title, project,
assignee, due, priority, confidence, needs_confirmation, questions.

## 5. Напоминания (3 сценария)

1. **Утро** (по времени юзера): задачи на сегодня по дедлайну + просроченные + важные.
2. **Вечер**: итог (сделано / не сделано / осталось) + задачи на завтра + просроченные + важные.
3. **По запросу** (ассистенту): «какие у меня задачи / важные / по проекту X» → ответ.

Канал доставки выбирает пользователь: **Telegram / push (PWA) / email (потом)**.
Можно несколько сразу. Дайджесты — cron в бэкенде (не n8n).

## 6. Авторизация — Telegram

- **Telegram Login Widget** в PWA → бэк проверяет HMAC-подпись по токену бота →
  находит/создаёт `users` через `auth_identities(provider=telegram, external_id=tg_id)` → JWT.
- **`users.id` — внутренний UUID, не Telegram.** Способ входа отвязан и сменяем.
- Fallback: вход по одноразовому коду из бота (если виджет капризит в in-app браузере).
- Mini App (позже) — вход автоматический по подписанному initData.
- Настройка: в @BotFather `/setdomain` = домен приложения.

## 7. Календарь

- Вид «календарь» в приложении: задачи с `dueAt`.
- Создание задачи из дня (полные пикеры — мы контролируем UI).
- ICS-фид (read-only) для подписки внешних календарей (Google/Apple) — Vikunja-модель
  «точка правды → витрина», но теперь фид генерим сами. См. `CALENDAR.md` (принцип).

## 8. Аналитика

- «На ком сколько задач» (по исполнителю × статус).
- По проектам: счётчики статусов, просрочка, доля выполнения.
- Доска проекта = канбан по статусам.

## 9. Внешние зависимости

**Ноль, кроме Telegram-бота.** Push — на самоподписанных VAPID (генерим сами).
БД/Redis/Qwen/HTTPS — на 158. Email (отправка/приём) — отложено.

## 10. Стек

- **Backend:** Node + TypeScript + **Hono** + PostgreSQL + **Drizzle ORM** + Zod + JWT.
  (Тот же стек, что FruitPortal — знаком команде.)
- **Frontend:** Next.js (App Router) **PWA**, mobile-first, устанавливается на экран,
  тёмная тема, свайпы статусов. Telegram Mini App — позже.
- **LLM:** существующий Gateway (FastAPI) + Qwen3:14b на GPU.
- **Инфра:** Caddy (HTTPS), Postgres, Valkey, всё в docker-compose на 158.

## 11. Модель данных

См. `backend/src/db/schema.ts`. Таблицы: `users`, `auth_identities`, `projects`,
`project_members`, `tasks`, `task_activity`, `push_subscriptions`.

## 12. Не в MVP (схема готова к добавлению)

Наблюдатели задач, повторяющиеся задачи, подзадачи/чек-листы, email-канал,
Telegram Mini App, GitHub Actions авто-деплой.

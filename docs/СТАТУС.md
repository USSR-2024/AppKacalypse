# AppKacalypse — статус работ

> Обновлено: 2026-06-13.
> Продуктовое ТЗ: `docs/ТЗ-app.md`. Инфраструктура/Gateway: `docs/ТЗ-appkacalypse.md`.

## Что это
AI-диспетчер задач для личной и командной работы. Задачи ставятся обычным языком
(приложение / встроенный чат-ассистент / Telegram-бот), локальный Qwen разбирает их
в структуру, система сама напоминает о сроках. **Свой бэкенд + PWA на своём GPU-сервере,
данные у себя, без внешних LLM-API. Единственная внешняя зависимость — Telegram-бот.**

## РАЗВОРОТ (2026-06-12)
Изначально строили поверх **Vikunja + n8n**. Отказались: Vikunja-UI «не вкусный» на мобиле,
AppFlowy (пробовали как альтернативу) — слабый API. **Решение: пишем свой продукт**, переиспользуя
готовое AI-ядро (LLM Gateway + Qwen), сервер, Caddy, HTTPS. Vikunja/n8n/AppFlowy выведены из продукта.

## Стек
- **Backend:** Node + TS + Hono + PostgreSQL + Drizzle + Zod + JWT. Контейнер `akc-backend:8081`.
- **Frontend:** Next.js 16 PWA (mobile-first, тёмная тема, ставится на экран). Контейнер `akc-frontend:3000`.
- **LLM:** FastAPI Gateway + `qwen3:14b` на GPU (контейнер `appkacalypse-llm-gateway`, `think:false`, ~5с).
- **Деплой:** бэк+фронт = compose-проект `akcapp` в общей сети `appkacalypse_default`; gateway — в проекте `appkacalypse`.
- **БД приложения:** отдельная база `akc` в `appkacalypse-postgres-1` (Vikunja-данные не трогаются).

## Готово и задеплоено
- ✅ **Бэкенд (S2):** Drizzle-схема (8 таблиц), JWT, CRUD задач/проектов/юзеров с фильтрами и правами, лог активности.

- ✅ **PWA (S3):** экраны Сегодня / Входящие (триаж) / Проекты (+ канбан-доска) / Календарь (повестка) / Профиль / Выполненные. Композер задачи (FAB), редактирование задачи по тапу, bottom-sheet пикеры.
- ✅ **AI-ассистент в приложении (S4):** вкладка «Ассистент» — чат, создание задач из текста (карточка-черновик) + **Q&A** («какие задачи на сегодня», «что важного», «что по проекту X»).
- ✅ **Telegram-бот (S5):** текст/пересылка → задача, доспрос 1 раунд, Q&A в телеге. Идентификация по telegram_id.
- ✅ **Напоминания (S5):** утренний/вечерний дайджесты в Telegram по времени и tz юзера (настройка в Профиле).

## Модель задачи
status (queued→in_progress→done / cancelled / archived), priority (low/normal/high) **и отдельно**
isImportant («важная, не забыть»), assignee, dueAt, source. Inbox = projectId null + isTriaged false.
Назначать задачи может любой любому; править — автор/исполнитель/admin.

## Что нужно от владельца
1. **Нажать Start в @appKACAlypse_bot** — иначе бот не сможет писать (ограничение Telegram): ни ответы, ни дайджесты.
2. Завести реальных пользователей (через их вход в приложение) и проекты — тогда ассистент будет резолвить имена в задачах.

## Известные нюансы
- `caddy reload` через exec не применяет конфиг — нужно `docker restart appkacalypse-caddy-1`.
- Дайджесты v1 — только Telegram-канал. Доспрос — один раунд, потом создаёт.
- Границы «сегодня» в дайджестах считаются по серверному времени (≈ Москва); точный per-user tz — на потом.

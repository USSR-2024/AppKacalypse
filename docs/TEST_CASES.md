# TEST_CASES — AppKacalypse

Минимум 50 сообщений (ТЗ §17). Файлы — в `test-data/messages/`.

## Категории
1. простая задача
2. несколько задач из одного сообщения
3. явный исполнитель
4. «мне / себе»
5. срок: «завтра / вечером / до пятницы / на следующей неделе»
6. проект указан явно
7. проект из контекста (по алиасам, см. справочники)
8. непонятный проект → уточнение
9. непонятный исполнитель → уточнение
10. задача с ограничением (constraints)
11. заметка (save_note)
12. напоминание (create_reminder)
13. календарное событие (create_calendar_event)
14. задача + событие (create_task_with_calendar_event)
15. сообщение без действия (no_action)
16. сообщение с секретом (redaction test)

## Redaction test
Вход: «Завтра проверить сервер root@1.2.3.4, токен sk-... не отправлять наружу».
Ожидание: текст обработан локально; в логах секреты замаскированы
(`[SERVER_LOGIN]`, `[API_KEY]`); задача создаётся после validation/confirmation.

## Формат файла сообщений
```json
{
  "category": "ru-basic",
  "cases": [
    {
      "id": "basic-01",
      "input": "Завтра Ивану проверить VPN сервер",
      "expect": {
        "intent": "create_task",
        "tasks": [{"assignee": "Иван", "project": "VPN", "due_text": "завтра"}]
      }
    }
  ]
}
```

## Smoke-tests (ТЗ §18)
**Базовые:** Vikunja открывается · n8n открывается · `ollama list` · Qwen на GPU ·
«Завтра Ивану проверить VPN сервер» → задача в Vikunja · бот подтвердил · в логах нет секретов.

**Календарь:** включить provider · «Завтра в 15:00 созвон с Иваном по VPN» → событие ·
«Ивану до пятницы проверить VPN» → задача, не событие · «В понедельник встреча по Hermes» → бот запросил время.

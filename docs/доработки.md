# Доработки AppKacalypse — пул задач (из тест-сессии Anna + Vitaly, 2026-06-14)

Источник: переписка тестирования. Решения по развилкам — согласованы с владельцем.

## Принятые архитектурные решения
- **Исполнители:** несколько на задачу, включая **внешних** (без аккаунта, имя текстом).
  Новая таблица `task_assignees` (taskId, userId **или** externalName). Миграция текущего `assigneeId`.
- **Контролёр:** один, поле `controllerId` на задаче. Если не выбран → = создатель.
  Контролёр видит задачу и может её менять. Сценарий внешнего: задачу делает не-член команды,
  а ответственный за результат — контролёр из команды.
- **Структура проектов:** разделы внутри проекта (секции а-ля Todoist), таблица `project_sections`.
- **Команды:** сущность `teams` + `team_members`; добавление команды в проект → её люди = участники.
- **Общение в задачах:** комментарии-тред `task_comments` + @упоминания → уведомление (push/Telegram).
  Полноценный мессенджер НЕ делаем (privacy + «не перегружать»).
- **История чата ассистента:** хранить 1 день, авто-очистка.
- **Десктоп:** отдельный расширенный layout на ≥lg (сайдбар, многоколоночность). Туда —
  аналитика и массовое создание задач. Мобилка остаётся «быстро поставить/не забыть/статус».

## Сквозной принцип
Не перегружать UI кнопками/вкладками/окнами. Любую фичу вписывать минималистично.

## Фаза 1 — быстрые баги/UX (низкий риск, фронт)
1. **Создание задачи молча падает** — у `TaskComposer.submit` нет catch; при ошибке модалка
   остаётся, юзер думает «не добавляется». Добавить тост ошибки + подсказку, что обязательно
   только Название.
2. **Кнопка «назад» закрывает форму** — системная back при открытом bottom-sheet уводит между
   страницами, форма остаётся. Перехватывать back → сначала закрыть модалку (все sheet: композер, пикеры).
3. **Подтверждение завершения** — «Завершить задачу?» вместо мгновенного тапа в кружок (TaskItem).
4. **«Выполненные» не найти** — экран `/done` есть, но его нет в нижней навигации. Дать доступ
   (ссылка в «Сегодня»/профиле или фильтр), не плодя вкладку.
5. **Создание проекта** — кнопка «+ Проект» → маленькая модалка (имя, цвет, участники/команда)
   вместо инлайн-поля. Разобраться с «плюсик не загорается» (воспроизвести на проде).

## Фаза 2 — модель «исполнители + контролёр + внешние» (схема + бэк + фронт + бот + ассистент)
6. Миграция: `task_assignees`, `controllerId`. Перенос `assigneeId`.
7. Бэк: видимость (исполнитель/контролёр/участник проекта видят), `canModify`, фильтры
   (`mine` = я среди исполнителей или контролёр), дайджесты.
8. Фронт: композер — мультивыбор исполнителей + пункт «указать внешнего» (поле, имя текстом);
   карточка задачи показывает исполнителей + контролёра.
9. Бот + ассистент: резолв нескольких исполнителей и внешних имён.

## Фаза 3 — карточка задачи
10. **Режим «только чтение» + кнопка «Редактировать»** — после создания название/описание не
    редактируются по тапу; правка по явной кнопке (избегаем случайных изменений).
11. **Комментарии-тред** в карточке + **@упоминания** → уведомление. Закрывает «работы»/«нюансы»
    (выгрузка 1С и т.п. крепится к задаче).
12. **Явный признак «личная / рабочая»** в карточке (сейчас только через селектор «Проект»).

## Фаза 4 — проекты и команды
13. Сущность **команды** (`teams`/`team_members`), добавление команды в проект.
14. **Разделы внутри проекта** (`project_sections`), переключение между ними.
15. **Командная видимость**: участники проекта видят ВСЕ его задачи (read), менять — только
    причастные (исполнитель/контролёр/creator/admin).
16. Экран **«Команда»** — видеть коллег, дёрнуть из задачи (@упоминание).
17. Доработать **триаж «Входящих»** (распределение по проектам/исполнителям).

## Фаза 5 — ассистент
18. **Редактирование существующих задач** через ассистента (сейчас всегда создаёт новую). ⚠️ нужны сценарии.
19. **Добавление работ к задаче** через ассистента (пересекается с комментами, п.11).
20. **История чата 1 день** (см. решения).
21. **Дизамбигуация исполнителя** — при нескольких подходящих («два Андрея») переспрашивать.

## Фаза 6 — конструкции задач
22. **Повторяющиеся задачи** (планёрки).
23. **Шаблоны задач** (а-ля Todoist: «встреча» → типовой набор).
24. **Массовое создание** из одного ввода (как протоколы — 10 задач разным исполнителям). В основном десктоп.

## Фаза 7 — десктоп и общее
25. **Десктоп-layout** (≥lg): сайдбар, многоколоночность.
26. **Аналитика/инфографика** — сколько задач выполнено, кто что делает (десктоп).
27. **Pull-to-refresh** — обновление потягиванием вниз.

## Фаза 8 — профиль/бот
28. **Редактирование ФИО** (`displayName`) в профиле; admin/owner — переименовать участников в `/users`.
29. **Отметка задачи выполненной из бота** — ⚠️ открыто (владелец склоняется к «бот только создаёт»).
